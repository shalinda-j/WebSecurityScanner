APT Tactics & Techniques

Back to APT Dashboard
What are APTs? Advanced Persistent Threats (APTs) are sophisticated, prolonged attack campaigns where adversaries gain and maintain unauthorized access to networks and systems. They typically use a variety of attack vectors and techniques to evade detection while extracting valuable data over extended periods.

APT Overview

Key Characteristics of APTs
  • Advanced - Utilizing sophisticated techniques and custom malware
  • Persistent - Long-term access and presence in target networks
  • Organized Actors - Often nation-states, criminal organizations, or well-funded groups
  • Targeted - Specific organizations or industries are chosen with clear objectives
  • Evasive - Designed to avoid detection by traditional security measures
Common APT Objectives
  • Data Theft - Intellectual property, customer data, financial information
  • Industrial Espionage - Stealing technological innovations and trade secrets
  • Geopolitical Advantage - Nation-state intelligence gathering
  • Financial Gain - Theft of financial assets or ransomware operations
  • Infrastructure Disruption - Critical systems sabotage

The APT Kill Chain

The "kill chain" describes the typical phases of an APT attack from initial reconnaissance to objective completion:

Phase 1 Initial Access
Phase 2 Persistence
Phase 3 Privilege Escalation
Phase 4 Lateral Movement
Phase 5 Data Exfiltration

{% if phase == 'initial_access' %}

This phase focuses on techniques used by attackers to gain their initial foothold in the target environment.

Spear Phishing

Targeted emails to specific individuals containing malicious attachments or links.

  • Often researched to appear legitimate and trustworthy
  • May impersonate colleagues, partners, or trusted organizations
  • Usually contain malicious office documents, PDFs, or executables
Defense: Email filtering, user awareness training, attachment sandboxing
Web Application Exploitation

Exploiting vulnerabilities in public-facing web applications to gain access.

  • SQL injection, cross-site scripting (XSS), and CSRF attacks
  • Exploitation of known vulnerabilities in web frameworks
  • Authentication bypass and insecure direct object references
Defense: Regular patching, WAF implementation, secure coding practices
Supply Chain Compromise

Targeting third-party vendors or software to compromise the primary target.

  • Compromise of software update mechanisms
  • Injecting malicious code into legitimate software packages
  • Using trusted relationships to bypass security controls
Defense: Vendor security assessments, code signing verification, SBOM practices
{% elif phase == 'persistence' %}

Once access is gained, attackers establish persistence to maintain their presence even if the initial access point is discovered and remediated.

Backdoor Installation

Creating hidden access mechanisms that can be used to regain access.

  • Web shells on compromised web servers
  • Custom remote access trojans (RATs)
  • Modification of existing services to include backdoor functionality
Defense: File integrity monitoring, application whitelisting, outbound connection monitoring
Credential Harvesting

Gathering valid credentials to maintain access even if the malware is detected.

  • Password hash dumping from memory
  • Keylogging and credential stealing malware
  • Exploitation of password storage mechanisms
Defense: Multi-factor authentication, privileged access management, credential monitoring
{% elif phase == 'privilege_escalation' %}

After establishing persistence, attackers seek to gain higher-level permissions to access more sensitive systems and data.

Permission Exploitation

Exploiting weaknesses in permission models to gain elevated access.

  • Misconfigured access control lists (ACLs)
  • Excessive user privileges or permission inheritance
  • Abuse of service accounts with elevated permissions
Defense: Principle of least privilege, regular privilege audits, just-in-time access
Vulnerability Exploitation

Exploiting known or zero-day vulnerabilities to gain elevated privileges.

  • Kernel exploits and privilege escalation vulnerabilities
  • DLL hijacking and application-specific vulnerabilities
  • Manipulation of unpatched internal services
Defense: Timely patching, vulnerability management, application sandboxing
{% elif phase == 'lateral_movement' %}

With elevated privileges, attackers move laterally across the network to expand their control and access to valuable assets.

Internal Reconnaissance

Mapping the internal network to identify valuable targets and movement paths.

  • Network scanning and enumeration
  • Active Directory enumeration and trust mapping
  • Service discovery and vulnerability mapping
Defense: Network segmentation, internal traffic monitoring, deception technology
Remote Service Exploitation

Using compromised credentials or exploits to access other systems.

  • Pass-the-hash and pass-the-ticket attacks
  • Remote execution via WMI, PowerShell, or PsExec
  • Exploitation of internal services (SMB, RDP, SSH)
Defense: Network segmentation, credential security, lateral movement detection
{% else %}

The final phase involves locating and extracting sensitive data from the target environment.

Data Collection

Identifying and collecting valuable data for extraction.

  • Database dumping and mass file collection
  • Email archive access and sensitive document gathering
  • Selective targeting based on keywords or file types
Defense: Data classification, encryption, access controls, data loss prevention
Data Exfiltration

Moving data out of the target environment through covert channels.

  • DNS tunneling and encrypted web channels
  • Steganography and custom protocols
  • Scheduled transfers during low-monitoring periods
Defense: Network monitoring, egress filtering, anomaly detection, DLP solutions

Defense Strategies Against APTs

Prevention
  • Email Security - Advanced filtering, attachment sandboxing
  • Vulnerability Management - Regular patching and updates
  • Network Segmentation - Limit lateral movement potential
  • Access Controls - Least privilege principle implementation
  • User Awareness - Security training and phishing simulations
Detection
  • Behavioral Analytics - Identify anomalous activities
  • Network Monitoring - Analyze traffic patterns and data flows
  • Endpoint Detection - Monitor for suspicious behavior
  • Deception Technology - Deploy honeypots and decoys
  • Threat Hunting - Proactively search for indicators of compromise
Response
  • Incident Response Plan - Documented procedures for breaches
  • Forensic Analysis - Detailed examination of compromised systems
  • Containment Strategies - Isolate affected systems
  • Threat Intelligence - Utilize IOCs to identify affected areas
  • Recovery Procedures - Restore systems from known-good backups
